```python
class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(10, 1)

    def forward(self, x):
        return self.fc(x)
```

---

### Step 4️⃣ Train the Model

```python
model = MyModel()
loss_fn = nn.MSELoss()
optimizer = optim.SGD(model.parameters(), lr=0.01)

for epoch in range(100):
    optimizer.zero_grad()
    output = model(torch.randn(5, 10))
    loss = loss_fn(output, torch.randn(5, 1))
    loss.backward()
    optimizer.step()
```

# Autograd in PyTorch

## What is Autograd?

**Autograd** is PyTorch’s **automatic differentiation engine**.

👉 It **automatically calculates gradients**  
👉 Used for **backpropagation**  
👉 Helps train neural networks

📌 **You do NOT need to calculate derivatives manually**

---

## Why Autograd is needed?

In deep learning, we need:

- Loss function
    
- Gradient of loss w.r.t. weights
    
- Update weights using gradient descent
    

Autograd does this **automatically**.

---

## Key Concept

> **Autograd tracks all operations on tensors and builds a computation graph.**

When you call:

```python
loss.backward()
```

PyTorch:

- Goes backward in the graph
    
- Computes gradients using **chain rule**
    

---

## `requires_grad` (Most Important 🔥)

```python
import torch

x = torch.tensor(2.0, requires_grad=True)
```

- `requires_grad=True` → track operations
    
- Gradient stored in `x.grad`
    

---

## Simple Autograd Example

```python
x = torch.tensor(3.0, requires_grad=True)
y = x ** 2
y.backward()

print(x.grad)
```

### Output:

```
6
```

Because:  
[  
y = x^2 \Rightarrow \frac{dy}{dx} = 2x = 6  
]

---

## Autograd with Multiple Operations

```python
x = torch.tensor(2.0, requires_grad=True)
y = x * 3
z = y ** 2
z.backward()

print(x.grad)
```

📌 Autograd applies **chain rule automatically**

---

## Gradients in Neural Networks

```python
loss.backward()

print(model.weight.grad)
```

- Gradients stored in `.grad`
    
- Used by optimizer to update weights
    

---

## Optimizer Step (Autograd + Optimizer)

```python
optimizer.zero_grad()   # clear old gradients
loss.backward()         # compute gradients
optimizer.step()        # update weights
```

📌 **Always reset gradients** (they accumulate)

---

## Computation Graph (Conceptual)

```
x → operations → loss
↑                 ↓
gradient ← backward
```

---

## Turning Off Autograd (Important ⚠️)

Used during **testing / inference**

```python
with torch.no_grad():
    output = model(x)
```

✔️ Saves memory  
✔️ Faster execution

---

## `.detach()` (Stop Gradient Tracking)

```python
y = x.detach()
```

- Removes tensor from computation graph
    
- No gradient calculated
    

---

## Common Mistakes

❌ Forgetting `requires_grad=True`  
❌ Forgetting `optimizer.zero_grad()`  
❌ Using in-place ops (`_`) incorrectly

---

## Autograd vs Manual Differentiation

|Manual|Autograd|
|---|---|
|Hard|Easy|
|Error-prone|Accurate|
|Slow|Fast|
|Not scalable|Scales well|

# 1️⃣ `requires_grad` (Very Important)

## What is `requires_grad`?

`requires_grad` is a **flag** in PyTorch that tells **Autograd**:

> “Track this tensor and calculate its gradient during backpropagation.”

---

## Why `requires_grad` is needed?

- Neural network **weights** must learn
    
- Learning requires **gradients**
    
- Gradients are computed **only if** `requires_grad=True`
    

---

## Example

```python
import torch

x = torch.tensor(2.0, requires_grad=True)
y = x ** 2
y.backward()

print(x.grad)
```

### Output:

```
4
```

📌 Because  
[  
y = x^2 \Rightarrow \frac{dy}{dx} = 2x = 4  
]

---

## When to use `requires_grad=True`?

|Tensor Type|requires_grad|
|---|---|
|Model weights|✅ True|
|Input data|❌ False|
|Labels|❌ False|
|Loss|Automatically handled|

---

## Turning off Gradient Tracking

### Using `no_grad()`

```python
with torch.no_grad():
    output = model(x)
```

### Using `detach()`

```python
y = x.detach()
```

✔️ Used during **testing / inference**

---

## Important Note ⚠️

Gradients **accumulate** by default.

```python
optimizer.zero_grad()
```

# 2️⃣ Backward Propagation (Backpropagation)

## What is Backward Propagation?

**Backward propagation** is the process of:

> Computing gradients of the loss function with respect to model parameters.

PyTorch does this using:

```python
loss.backward()
```

---

## Why Backpropagation is needed?

- To **minimize loss**
    
- To **update weights**
    
- To **train neural networks**
    

---

## Backpropagation Steps

### 1️⃣ Forward Pass

```python
output = model(x)
loss = loss_fn(output, target)
```

### 2️⃣ Backward Pass

```python
loss.backward()
```

### 3️⃣ Weight Update

```python
optimizer.step()
```

### 4️⃣ Reset Gradients

```python
optimizer.zero_grad()
```

---

## Backward Propagation Example

```python
x = torch.tensor(3.0, requires_grad=True)
y = x ** 3
y.backward()

print(x.grad)
```

### Output:

```
27
```

Because:  
[  
\frac{dy}{dx} = 3x^2 = 27  
]

---

## Computation Graph (Concept)

```
Input → Forward Pass → Loss
   ↑                    ↓
Gradients ← Backward Pass
```

---

## Key Differences

|Forward Pass|Backward Pass|
|---|---|
|Compute output|Compute gradients|
|Uses data|Uses loss|
|No gradients|Gradients stored|

---

## Common Mistakes (Exam Alert ⚠️)

❌ Forgetting `loss.backward()`
