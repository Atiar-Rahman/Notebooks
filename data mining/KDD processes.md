# 🔹 **KDD (Knowledge Discovery in Databases)**

👉 KDD is the **overall process** of extracting useful knowledge from data.  
👉 **Data Mining** is only **one step** within KDD.

---

## ✅ **Steps of KDD**

1. **Data Cleaning** 🧹
    
    - Remove noise, errors, missing values, duplicates.
        
    - Example: Fixing wrong ages like `-5` or removing incomplete records.
        
2. **Data Integration** 🔗
    
    - Combine data from multiple sources into a single, consistent dataset.
        
    - Example: Merging sales data (Excel) + customer details (database).
        
3. **Data Selection** 🎯
    
    - Select only relevant data needed for the task.
        
    - Example: From hospital data, selecting only “patients with diabetes” for analysis.
        
4. **Data Transformation** 🔄
    
    - Convert data into a suitable format for mining.
        
    - Methods: Normalization, encoding, dimensionality reduction.
        
    - Example: Scaling income values between `0 and 1`.
        
5. **Data Mining** 💎
    
    - Apply algorithms to discover patterns.
        
    - Techniques: Classification, clustering, regression, association rules.
        
    - Example: Finding that “customers aged 20–30 often buy energy drinks.”
        
6. **Pattern Evaluation** ✅
    
    - Identify truly useful and interesting patterns (not random).
        
    - Methods: Interestingness measures, significance testing.
        
    - Example: Ignore patterns that occur by chance.
        
7. **Knowledge Presentation** 📊
    
    - Present the discovered knowledge in a human-friendly way.
        
    - Tools: Visualization, reports, dashboards.
        
    - Example: Showing customer segments in a chart for marketing decisions.
        

---

## 🔄 **Flow of KDD**

**Raw Data → Cleaning → Integration → Selection → Transformation → Data Mining → Pattern Evaluation → Knowledge**

---

✨ In short:

- **KDD = Big umbrella process**
    
- **Data Mining = Core step inside KDD**
