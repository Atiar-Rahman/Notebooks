## feature engineering and data `Preprocessing` for data mining

1. Objectives
**goal:** To learn how to clean, `preprocess`, and  engineer features for better model performance. 

*Example*
Before training any ml model, you must:
- Handle missing value
- remove noise and outliers
- Normalize or scale data,
- encode categorical variables
- create  a new meaning features

These steps improve accuracy and `interpretability`.

2. problem analysis:
Feature engineering and `preprocessing` are  fundamental. 
This  lab uses the california housing dataset to perform
- cleaning(remove duplicates, handle missing data)
- transformation(scaling, encodig, discentization)
- feature engineering (new columns)
- pca(reduce dimenstion)
- model evaluation( before vs after feature engineering)


3. learning outcome
you will learn to:
- handle missingg/noisy/inconsistent data
- transform features(scale, normalize, discreize)
- create new derived attributes
- compare model performance before and after  feature engineering.
