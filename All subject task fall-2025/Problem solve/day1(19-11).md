# ✅ **Python Solution (32-bit safe reversal)**

The key idea:

- Pop the last digit using `x % 10` (or `x % -10` for negatives).
    
- Push the digit into the reversed number.
    
- Before pushing, **check overflow** using integer math only.
    

32-bit range:

```
INT_MIN = -2^31 = -2147483648
INT_MAX =  2^31-1 =  2147483647
```

---

# ✔ **Working Code (Python)**

```python
def reverse(x: int) -> int:
    INT_MAX = 2**31 - 1
    INT_MIN = -2**31

    rev = 0
    sign = -1 if x < 0 else 1
    x = abs(x)

    while x != 0:
        pop = x % 10
        x //= 10

        # Check overflow BEFORE multiplying
        if rev > INT_MAX // 10 or (rev == INT_MAX // 10 and pop > 7):
            return 0
        
        rev = rev * 10 + pop

    return sign * rev
```

---

# 🧪 Examples

### ➤ `reverse(123)` → `321`

### ➤ `reverse(-123)` → `-321`

### ➤ `reverse(120)` → `21`

### ➤ `reverse(1534236469)` → `0` (Overflow)

---

# 📝 Explanation (Simple)

```
pop = last digit
rev = rev * 10 + pop
```

But before multiplying, we check:

```
rev > INT_MAX // 10     → overflow
rev == INT_MAX // 10 and pop > 7 → overflow
```

7 is the last digit of INT_MAX (2147483647).

→ **O(n) time**, because each character is visited at most twice.

# ✅ **Python Solution (Sliding Window)**

```python
def lengthOfLongestSubstring(s: str) -> int:
    char_index = {}
    left = 0
    max_len = 0

    for right, ch in enumerate(s):
        if ch in char_index and char_index[ch] >= left:
            # Move left pointer past the last duplicate
            left = char_index[ch] + 1
        
        char_index[ch] = right
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

---

# 🧠 Explanation (Simple)

We keep a window `[left, right]` that contains **no duplicate characters**.

- When we see a duplicate character at `right`:  
    → Move `left` just after the previous occurrence.
    
- Update the last index of each character in `char_index`.

# ✅ **Python Solution (Binary Search Partition Method)**

```python
def findMedianSortedArrays(nums1, nums2):
    # Ensure nums1 is the smaller array
    if len(nums1) > len(nums2):
        nums1, nums2 = nums2, nums1

    m, n = len(nums1), len(nums2)
    total = m + n
    half = total // 2

    left, right = 0, m

    while True:
        i = (left + right) // 2   # partition for nums1
        j = half - i              # partition for nums2

        # Left and right elements around partition
        Aleft = nums1[i-1] if i > 0 else float('-inf')
        Aright = nums1[i] if i < m else float('inf')
        Bleft = nums2[j-1] if j > 0 else float('-inf')
        Bright = nums2[j] if j < n else float('inf')

        # Valid partition check
        if Aleft <= Bright and Bleft <= Aright:
            # If total is odd
            if total % 2:
                return min(Aright, Bright)
            # If even
            return (max(Aleft, Bleft) + min(Aright, Bright)) / 2

        # Move search window
        elif Aleft > Bright:
            right = i - 1
        else:
            left = i + 1
```

---

# 🧠 **Explanation (Simple)**

We want to split both arrays into **left half** and **right half** such that:

```
left side   <=   right side
```

Example partition:

```
nums1: [1 | 3]
nums2: [2 | 4, 5, 6]
```

Left half values: `1, 3, 2`  
Right half values: `4, 5, 6`

If the partition is correct, the median is:

- max of left side (only if we have odd elements)
    
- average of max-left and min-right (if even elements)
    

We use **binary search** to adjust the cut point in nums1 until:

```
Aleft <= Bright  AND  Bleft <= Aright
```

This ensures a correct partition.

---

# 🧪 **Examples**

### Example 1:

nums1 = [1, 3]  
nums2 = [2]

Merged = [1, 2, 3]  
Median = **2**

### Example 2:

nums1 = [1, 2]  
nums2 = [3, 4]

Merged = [1, 2, 3, 4]  
Median = **(2 + 3) / 2 = 2.5**

# ✅ **Python Solution: Expand Around Center**

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        if not s or len(s) == 1:
            return s
        
        start = 0
        end = 0
        
        def expand(l, r):
            while l >= 0 and r < len(s) and s[l] == s[r]:
                l -= 1
                r += 1
            return r - l - 1   # length of palindrome
        
        for i in range(len(s)):
            len1 = expand(i, i)       # odd-length palindrome
            len2 = expand(i, i + 1)   # even-length palindrome
            max_len = max(len1, len2)
            
            if max_len > end - start:
                start = i - (max_len - 1) // 2
                end = i + max_len // 2
        
        return s[start:end + 1]
```

---

# 🧠 Explanation (Simple)

Every palindrome is centered at either:

- **one character** → odd length (like "bab")
    
- **two characters** → even length (like "bb")
    

For each index `i`, we expand outward from:

```
(i, i)     → odd length
(i, i+1)   → even length
```

We track the longest span (`start` to `end`) found.

---

### Example 1

Input: `"babad"`  
Possible palindromes: `"bab"`, `"aba"`  
Output: **"bab"**

### Example 2

Input: `"cbbd"`  
Longest palindrome: **"bb"**
