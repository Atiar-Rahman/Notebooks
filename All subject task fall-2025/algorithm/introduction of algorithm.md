An **algorithm** is a **step-by-step procedure or set of rules** designed to perform a specific task or solve a particular problem.

### 🔹 Simple Definition:

An **algorithm** is like a **recipe** — it tells you what steps to take and in what order to get a desired result.

---

### 🔹 Example:

Suppose you want to find the largest number in a list of numbers.  
An algorithm for this could be:

1. Start with the first number as the largest.
    
2. Compare it with the next number.
    
3. If the next number is larger, make it the new largest.
    
4. Repeat until all numbers have been compared.
    
5. The last “largest” number is the answer.
    

---

### 🔹 Characteristics of a Good Algorithm:

1. **Input:** Takes some input values.
    
2. **Output:** Produces at least one output.
    
3. **Definiteness:** Each step is clear and unambiguous.
    
4. **Finite-ness:** Has a finite number of steps (it must end).
    
5. **Effectiveness:** Each step must be basic enough to be done by hand or by a computer.
    

---

### 🔹 Real-life Examples:

- Sorting a list of names alphabetically.
    
- Searching for a word in a document.
    
- Finding the shortest route in Google Maps.
    
- Logging into your email account (authentication algorithm).

Here are the **main types of algorithms** you should know 👇

---

### 🔹 **1. Search Algorithm**

Used to **find an element** from a collection of data.  
**Examples:**

- **Linear Search:** checks each element one by one.
    
- **Binary Search:** divides the data in half each time (works only on sorted data).
    

🧠 _Use when you need to locate a specific item quickly._

---

### 🔹 **2. Sorting Algorithm**

Used to **arrange data** in a specific order (ascending or descending).  
**Examples:**

- **Bubble Sort**
    
- **Selection Sort**
    
- **Insertion Sort**
    
- **Merge Sort**
    
- **Quick Sort**
    

🧠 _Use for organizing data before searching or analysis._

---

### 🔹 **3. Recursive Algorithm**

An algorithm that **calls itself** repeatedly until a condition is met.  
**Example:**  
Factorial of a number →  
`factorial(n) = n * factorial(n - 1)`

🧠 _Used in tree traversal, backtracking, and divide-and-conquer problems._

---

### 🔹 **4. Divide and Conquer Algorithm**

Breaks a problem into **smaller sub-problems**, solves each, and then **combines** the results.  
**Examples:**

- Merge Sort
    
- Quick Sort
    
- Binary Search
    

🧠 _Efficient for large problems that can be split into smaller ones._

---

### 🔹 **5. Dynamic Programming Algorithm**

Solves problems by **breaking them down into overlapping subproblems** and **storing results** to avoid repetition.  
**Examples:**

- Fibonacci sequence
    
- Shortest path (like Bellman-Ford)
    

🧠 _Great for optimization problems._

---

### 🔹 **6. Greedy Algorithm**

At each step, it **chooses the best immediate option** (local optimum) hoping it leads to the best overall result (global optimum).  
**Examples:**

- Kruskal’s Algorithm (Minimum Spanning Tree)
    
- Dijkstra’s Algorithm (Shortest Path)
    

🧠 _Fast and simple for problems that have an optimal greedy choice._

---

### 🔹 **7. Backtracking Algorithm**

Tries all possible solutions and **goes back** when a path fails.  
**Examples:**

- N-Queens Problem
    
- Sudoku Solver
    
- Maze solving
    

🧠 _Used when exploring multiple possibilities (like puzzles)._

---

### 🔹 **8. Brute Force Algorithm**

Tries **all possible solutions** until the correct one is found.  
**Example:**

- Guessing a password by testing every combination.
    

🧠 _Simple but inefficient for large problems._

---

### 🔹 **9. Randomized Algorithm**

Uses **random numbers** to decide the next step in the process.  
**Examples:**

- Randomized Quick Sort
    
- Monte Carlo Algorithm
    

🧠 _Used in probabilistic and optimization problems._
